## Brand Voice & Tone

### Writing Style
- **Bold & Confident** — Headlines are punchy and declarative ("Proven Global Success", "Every Conversation is an Experience")
- **Character-Driven** — Always refer to the mascot by name (Liza, Bastiaan, Fin, Woof), never as "the chatbot" or "the system"
- **Warm but Professional** — Conversational without being casual. We speak like a trusted creative partner, not a SaaS vendor
- **No excessive jargon** — Avoid terms like "NLP pipeline", "inference layer". Instead say "Empathy Engine", "Knowledge Base", "Brand Agent"
- **CTA phrasing** — Use "Book a call" or "Book now" (implies scarcity/premium), never "Try free" or "Sign up"

## Slide-Specific Content Logic

### P3 — Pain Points
AI generates 4 pain cards based on `STATE.industry` and `STATE.pain`. Each card has:
- Emoji icon
- Short title (max 4 words)
- Description (1-2 sentences)
Pain should feel specific to the client's world, not generic.

### P7 — Personality & Empathy
Content structure (5 points):
1. **Positioning**: {{Mascot_Name}} is not a search box, but a digital colleague with character and empathy
2. **Co-development**: Through a workshop, we define {{Mascot_Name}}'s personality traits together
3. **Empathy Engine**: {{Mascot_Name}} recognizes user emotions and responds accordingly
4. **Scenario example**: Industry-specific frustration scenario (auto-generated)
5. **Brand translation**: We translate your brand values into a consistent dialogue style

### P9 — Chatflow
Dynamic flowchart showing:
1. Welcome message → 2. User input → 3. Processing (Emotion detection, Knowledge search, Control Agent) → 4. Compound response (Empathy + Info + Upsell) → 5. Conversion path

All dialogue examples must match the client's industry and pain points.

3-column layout. Categories auto-generated based on industry:
- Each column has a Flow name + 5 bullet items
- Each column has a Documents section + 5 document names

## Technical Architecture

### Overview
- **Pure frontend** HTML+JS single-file app (`index.html`, ~2500+ lines), no build step
- Served via `python3 -m http.server 8080 --directory auto-proposal-gen`
- Claude Anthropic API calls use `anthropic-dangerous-direct-browser-access: true`, model: `claude-sonnet-4-20250514`

### Core Data Flow
```
User fills form → genAll() → scrape client website (allorigins proxy)
→ build prompt with client info + website text
→ Claude API returns JSON (cover_headline, pain_cards, feature_items, chat messages, ty_body)
→ parsed into STATE → rebuildSlides() → 16-slide deck rendered
```

### Key Functions
| Function | Purpose |
|----------|---------|
| `buildSlideHTML(sid)` | Giant switch statement generating slide HTML via template literals |
| `rebuildSlides()` | Regenerates all active slides, adds logo overlay |
| `scaleCanvas()` | Scales 1920×1080 canvas to fit viewport |
| `goTo(i)` | Navigates to slide index |
| `genAll()` | Orchestrates AI content generation |
| `researchClientWebsite(url)` | Fetches client website via allorigins.win proxy |
| `tx(key, lang)` | Translation lookup from TX object |
| `setLayout(name)` | Switches between 'spread', 'impact', 'editorial' layouts |
| `mascotImgTag(sid, V)` | Returns img tag for uploaded mascot image |
| `alpha(color, opacity)` | Helper for rgba colors |

### State Management
- `STATE` object holds all app state (client, industry, colors, font, lang, layout, mascotStyle, AI results...)
- `TX` object contains all translatable strings (7 languages: en, nl, zh, de, fr, ja, es)
- `ALL_SLIDES[]` defines 16 slides with id, title, emoji, quick flag
- `MASCOTS[]` defines 4 mascot characters (bastiaan, mieke, liza, jamal)
- `RM_MS[]` defines 6 roadmap milestones with multilingual labels
- Layout variants: `STATE.layout` = 'spread' | 'impact' | 'editorial'

### Slide Theming
- Light slides (s-mopt, s-mdesign, s-emotion, s-kb): use `.sl-bg-light` with dark text
- Dark slides: use brand colors as background with light text
- CSS vars: `--B1`, `--B2`, `--BG`, `--TX`, `--fd` (display font), `--fb` (body font)
- Logo overlay: white on dark slides, black on light slides (logic in rebuildSlides)

### Filmstrip
- Mini slide thumbnails below main display (like PowerPoint)
- Uses `buildSlideHTML` at 0.0667 scale (1920px → 128px)
- Docks directly below slide stage, no scroll needed

### Important Editing Notes
- **Template literals**: The file uses backticks and `${}` extensively. The Edit tool often fails on these patterns. Use Python patch scripts instead:
  ```python
  with open('index.html','r') as f: src=f.read()
  src = src.replace('old pattern', 'new pattern')
  with open('index.html','w') as f: f.write(src)
  ```
- **TX translations**: Always add new translatable strings to the TX object and use `tx('key', L)` in templates
- **Industry adaptation**: Use `STATE.industry.toLowerCase().includes()` checks for: retail, healthcare, dental, education, office, wtc, legal, finance
